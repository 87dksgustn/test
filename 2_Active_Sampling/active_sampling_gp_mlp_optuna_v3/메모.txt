CV로 계산하는 지표
분류:
accuracy: 전체 분류 정확도
tp_precision: TP라고 예측한 것 중 실제 TP 비율
tp_recall: 실제 TP를 얼마나 놓치지 않고 찾았는지
tp_f1: TP precision/recall의 균형 점수
회귀 (NoTP 조건에서 Tmax):
tmax_mae: 평균 절대 오차
tmax_rmse: 큰 오차에 더 민감한 제곱근 오차
tmax_r2: 설명력
안정화 점수:
weighted_mean_score: 가중 평균 점수
weighted_std_score: 가중 표준편차
stable_score = weighted_mean_score - CV_STD_PENALTY x weighted_std_score


- Recall = TP / (TP + FN)
실제 TP 중에서 모델이 TP라고 제대로 찾아낸 비율
 : 놓치면 안 되는 대상을 얼마나 잘 잡는지 (실제 관심 대상을 놓치면 안 됨)

- Precision = TP / (TP + FP)
모델이 TP라고 예측한 것 중에서 실제 TP인 비율
 : 모델의 TP 예측을 얼마나 믿을 수 있는지 (Positive 예측이 틀리면 비용이나 위험이 큼)

- F1 = (2 x Precision x Recall) / (Precision + Recall)
Precision과 Recall의 조화평균
 : Precision과 Recall 중 어느 하나만 높이는 모델을 피하고, 두 지표가 적절히 균형을 이루는지 평가
   ex) 모델이 거의 모든 데이터를 TP라고 예측하면 -> Recall은 높을 수 있음 but, 실제 NoTP도 TP라고 예측하므로 Precision은 낮아짐
       반대로 아주 확실한 데이터만 TP라고 예측하면 -> Preicision은 높아짐 but, 실제 TP를 많이 놓치므로 Recall은 낮아짐
       이렇게 양 극단을 모두 불리하게 평가

- Accuracy = (TP + TN) / (TP + TN + FP + FN)
전체 예측 중 맞춘 비율
but 데이터가 불균형하면 Acc만으로는 모델을 제대로 평가하기 어려움
   ex) TP 500개 / NoTP 50개 데이터에서 모든 데이터를 Pass로 예측하면 Acc=95%가 나오지만 실제 fail을 하나도 찾지 못함

*TP 를 positive로 놓은 이유 : Fail Recall을 통해 실제 위험한 설계를 얼마나 안 놓치는지를 중점적으로 보기 위함

Recall > 0.95 라는 안전 조건을 먼저 만족시킨 하에 F1 > 0.95인 모델을 선택하는 전략
